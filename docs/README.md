# Documentation Hub

A centralized documentation repository organized for clear structure and easy navigation.

## Overview

This repository serves as a documentation hub containing structured Markdown files and images. All documentation content is organized within the `docs` folder, while other folders are reserved for non-documentation purposes (LLMs, scripts, tools, etc.).

## Repository Structure

```
documentation-hub/
├── docs/                     # All documentation content
│   ├── README.md            # This file - documentation overview
│   ├── getting-started/     # Getting started guides
│   ├── guides/              # How-to guides and tutorials
│   ├── reference/           # Technical reference documentation
│   ├── architecture/        # System architecture documentation
│   ├── api/                 # API documentation
│   ├── images/              # Documentation images (.jpeg format)
│   └── templates/           # Documentation templates
├── tools/                   # Non-documentation: Development tools
├── scripts/                 # Non-documentation: Automation scripts
├── llms/                    # Non-documentation: LLM-related files
├── .gitignore              # Git ignore rules
└── README.md               # Repository overview
```

## Documentation Guidelines

### File Types Allowed in `docs/` Folder
- **Markdown files**: `.md` format for all documentation
- **Images**: `.jpeg` format only for visual documentation
- **Configuration**: `.gitignore` for version control rules

### Organization Principles
1. **Clear Structure**: Logical folder hierarchy for easy navigation
2. **Consistent Naming**: Use lowercase with hyphens for folder/file names
3. **Markdown Standard**: All text documentation in Markdown format
4. **Image Standards**: JPEG format for all documentation images
5. **Version Control**: Proper gitignore configuration

## Getting Started

1. Navigate to the appropriate subfolder in `docs/`
2. Follow the existing naming conventions
3. Use Markdown for all text documentation
4. Place images in the `docs/images/` folder
5. Reference images using relative paths

## Contributing

When adding documentation:
- Maintain the folder structure
- Use descriptive file names
- Follow Markdown best practices
- Keep images in JPEG format
- Update this README if adding new sections
